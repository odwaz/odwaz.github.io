Wallet Microservices

A distributed digital wallet platform — five Spring Boot services behind an API gateway, fully reactive with WebFlux and R2DBC. OAuth2 + OTP authentication, wallet and transaction domains split into independent services, and webhook notifications — shipped through a GitOps pipeline onto Terraform-provisioned AWS.

What it does

oauth2 + otp auth

Dedicated auth service issuing JWTs with expiration and validation, OAuth2 grant types, and one-time-password verification in the login flow.

wallet domain

Wallet and transaction concerns split into separate services, so value movement and account state scale and deploy independently.

webhook notifications

A dedicated service pushes wallet and transaction events to consumers — async patterns keep the core flow non-blocking.

single entry point

The API gateway owns routing and token validation, so services stay internal and the public surface stays small.

Engineering highlights

The platform is as much about how it ships as what it serves.

GitOps delivery with ArgoCD

GitLab CI/CD builds and tests; ArgoCD syncs the desired state from Git to the cluster. Conditional workflows skip unchanged services — measured ~80% faster pipelines.

Infrastructure as code

Terraform provisions the AWS footprint (ECR, ECS) — environments are reproducible, reviewable, and destroyable, with multi-environment deployment across dev, staging, and production.

Security scanning in the pipeline

Trivy scans images before they ship; configuration is environment-based with no hardcoded secrets; logging is scrubbed of sensitive data.

Reactive, stateless services

WebFlux + Project Reactor with R2DBC — non-blocking I/O for high concurrency, stateless services ready for horizontal scaling behind a load balancer.

Fast, cache-aware builds

Docker BuildKit multi-stage builds with layer caching — measured ~3× faster image builds — and SonarCloud quality gates on every merge.
